## 待实现的需求

### 需求 1: 按层级区分 Enter/Tab 行为

**脑图层级定义：**
```
测试用例（虚拟根）
├── 登录功能（第1层 = 功能模块，无子列数据）
│   ├── 登录成功（第2层 = 测试用例，有子列数据）
│   │   ├── 用例描述（第3层 = 字段名，不可编辑）
│   │   │   └── 输入正确账户（第4层 = 字段值，可编辑）
│   │   ├── 前置条件
│   │   │   └── 打开登录页面
│   │   └── ...
│   └── 登录失败（第2层）
└── 首页（第1层）
```

**操作规则：**

| 选中节点层级 | Enter 操作 | Tab 操作 |
|-------------|-----------|---------|
| 第1层（功能模块） | 新增同级功能模块（只写第1列，不展开字段） | 新增测试用例（写第1列 + 关联父记录 + 展开所有字段） |
| 第2层（测试用例） | 新增同级测试用例（写第1列 + 关联父记录 + 展开所有字段） | 不操作（字段节点由数据驱动，不手动添加） |
| 第3层（字段名） | 不操作 | 不操作 |
| 第4层（字段值） | 不操作 | 不操作 |

**实现思路：**
1. 判断节点层级：通过节点 ID 格式区分
   - 记录节点: ID = recordId（如 `recvkdzbLCTXUO`）
   - 字段名节点: ID = `{recordId}_{fieldId}`
   - 字段值节点: ID = `{recordId}_{fieldId}_val`
2. 判断记录节点是第1层还是第2层：看它在 records 中是否有 parentFieldId 值
   - 有父记录 → 第2层（测试用例）
   - 无父记录 → 第1层（功能模块）
3. 拦截 jsMind 的默认 addchild/addbrother 行为，替换为自定义逻辑
4. 新增测试用例时，创建记录后立即刷新脑图（让字段节点自动生成）

### 需求 2: 编辑字段值后正确回写

- 双击第4层节点（字段值）→ 编辑 → 回车确认 → 更新多维表格对应行的对应列
- 双击第2层节点（测试用例名称）→ 编辑 → 回车确认 → 更新多维表格对应行的第1列
- 第1层和第3层节点编辑后也要正确回写

### 需求 3: 样式优化

- 第3层（字段名）节点样式区分：更小字号、灰色文字，表示不可手动添加
- 第4层（字段值）节点样式：可编辑提示（如虚线边框或浅色背景）
- 连线保持蓝色曲线

## 技术要点

- jsMind 事件类型: show=1, resize=2, edit=3, select=4
- jsMind edit 事件的 evt 值: `add_node`, `insert_node_after`, `insert_node_before`, `update_node`, `remove_node`, `move_node`
- 飞书关联字段(type 18)写入格式: `table.setCellValue(fieldId, recordId, { recordIds: [targetId] })`
- 飞书单选字段(type 3)写入格式: `{ text: '选项文本' }`
- 飞书文本字段写入格式: `[{ type: 'text', text: '内容' }]`
